## 1. 项目一句话说明

这是一个以无限画布为核心的 AI 可视化工作流应用。  
用户通过拖拽节点、连接节点、填写参数，完成：

- 提示词生成
- 文生图 / 图生图
- 图生视频

当前主线路已经明确：

- 真实能力以火山引擎 Ark 为主
- ComfyUI 不是当前优先事项
- Mock 适配器仅作为兜底和本地离线调试用途

## 4. 当前系统运行原理

这一节非常重要，接手人先看这一节，基本就能理解项目为什么能跑。

### 4.1 前端如何触发生成

以图片生成为例：

1. 用户在 `imageGen` 节点填写 prompt 和参数
2. 前端先根据节点 id 建立 WebSocket 监听
3. 前端调用后端生成接口：
   - 图片：`POST /api/generate`
   - 视频：`POST /api/generate/video`
4. 后端异步创建任务并执行
5. 后端通过 `/ws/progress/{node_id}` 持续推送进度
6. 前端更新节点状态、进度和输出结果
7. 结果同步到下游节点

相关文件：

- 前端：
  - `frontend/src/services/nodeGeneration.ts`
  - `frontend/src/services/videoGeneration.ts`
  - `frontend/src/services/websocket.ts`
- 后端：
  - `backend/app/api/generate.py`
  - `backend/app/api/ws.py`
  - `backend/app/services/task_service.py`

### 4.2 后端如何选择适配器

后端通过 `TaskService` 决定最终使用哪个适配器：

- `volcengine`
- `comfyui`
- `mock`

当前推荐与默认配置：

- 默认适配器：`volcengine`
- 若火山引擎配置正常，主链路就走真实能力
- `mock` 仅用于兜底调试

关键文件：

- `backend/app/services/task_service.py`
- `backend/app/adapters/registry.py`
- `backend/app/main.py`

### 4.3 图片上传如何参与后续生成

上传接口：

- `POST /api/assets/upload`

上传后，文件会保存到：

- `backend/data/uploads/`

前端上传成功后，会拿到 `/uploads/xxx.png` 这样的地址；  
后续在图生图、图生视频时，后端会将该地址解析为本地文件，再转成可被火山引擎消费的输入。

关键文件：

- `backend/app/api/assets.py`
- `backend/app/adapters/volcengine_adapter.py`

### 4.4 提示词生成如何工作

提示词优化是单独一条后端链路，不和图片/视频生成混在一起：

- 前端调用：`POST /api/prompts/generate`
- 后端通过 `PromptGenerationService` 调用火山引擎对话模型
- 返回一个整理后的生产可用 prompt
- 前端将返回结果直接回填到当前节点的 prompt 输入框

关键文件：

- `frontend/src/services/promptGeneration.ts`
- `backend/app/api/prompts.py`
- `backend/app/services/prompt_service.py`

### 4.5 一键执行如何工作

工作流执行并不是“并发乱跑”，而是按拓扑顺序执行：

1. 先根据连线关系做拓扑排序
2. 仅执行可执行生成节点：
   - `imageGen`
   - `videoGen`
3. 上游完成后，下游再执行

关键文件：

- `frontend/src/services/workflowRunner.ts`
- `frontend/src/utils/workflowExecution.ts`

## 6. 本地运行方法

### 6.1 后端

```powershell
cd backend
pip install -r requirements.txt
python run.py
```

启动后检查：

- `http://localhost:8000/docs`
- `http://localhost:8000/api/health`

建议确认 `/api/health` 返回中包含：

- `volcengine_enabled: true`
- `default_adapter: "volcengine"`
- `adapters` 中包含 `volcengine`

### 6.2 前端

```powershell
cd frontend
npm install
npm run dev
```

访问：

- `http://localhost:3000`

### 6.3 特别注意

如果修改了 `backend/.env`，必须重启后端进程。  
否则运行中的旧进程不会读取到新配置。

## 7. 当前关键环境变量

后端主链路依赖：

```env
DEFAULT_ADAPTER=volcengine
VOLCENGINE_ENABLED=True
ARK_API_KEY=你的火山引擎密钥
VOLCENGINE_REQUEST_TIMEOUT=180
VOLCENGINE_VIDEO_TIMEOUT=900
VOLCENGINE_POLL_INTERVAL=3
```

说明：

- `VOLCENGINE_REQUEST_TIMEOUT`：单次 HTTP 请求超时
- `VOLCENGINE_VIDEO_TIMEOUT`：视频任务轮询总超时
- 图生视频如果后续仍偏慢，可继续增大 `VOLCENGINE_VIDEO_TIMEOUT`
