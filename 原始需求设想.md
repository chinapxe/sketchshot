为了让你的 AI 开发组能够高效进入实质性开发，我们需要将之前的概括方案转化为**可执行的工程手册**。

这份细化方案将项目拆解为四个核心模块：**画布交互层**、**逻辑编排层**、**AI 算力层**、以及**数据持久化层**。

---

## 1. 核心功能需求细化 (User Stories & Features)

### A. 画布交互 (Frontend Experience)
* **节点操作：** 支持拖拽添加、连接、克隆、禁用/启用节点。
* **智能连线：** 连线具有类型检查（例如：Image 输出只能连 Image 输入，不能连 Text 输入）。
* **预览与缩放：** 每个节点内置 Canvas 预览区，支持点击放大查看生成的高清大图或视频。
* **参数热更新：** 修改节点参数（如 Prompt）后，下游受影响的节点显示“待更新”状态。

### B. AI 工作流逻辑 (Workflow Logic)
* **角色固化 (Identity Locking)：** 提供“全局角色节点”，提取上传图片的特征码（Embedding），强制注入下游所有 T2I 节点。
* **分支对比：** 用户可以从一个角色出发，拉出两个文生图节点，分别测试不同风格（如“古风” vs “赛博朋克”）。
* **中断与恢复：** 长时间生成视频任务支持进度保存，防止页面刷新导致任务丢失。

---

## 2. 深度技术选型 (Technical Deep-Dive)

### 前端：Canvas & Logic
* **框架：** **React 18 + Vite** (构建速度快)。
* **组件库：** **React Flow Pro** (如果预算允许) 或 **React Flow**。
* **状态引擎：** **XState**。*理由：* 画布节点的状态（空闲、排队、生成中、成功、失败）非常复杂，用状态机管理能极大地减少 Bug。
* **实时通信：** **Socket.io**。用于接收后端推送的生成进度条和中间采样图。

### 后端：Orchestration & Queue
* **框架：** **FastAPI** (异步性能高)。
* **引擎对接：** **ComfyUI-Manager + API**。通过 WebSocket 协议操控 ComfyUI 执行。
* **任务分发：** **Redis Streams**。比 Celery 更轻量，适合处理具有前后依赖关系的流式任务。
* **媒体处理：** **PyAV / FFmpeg-python**。用于分镜图到视频的快速合成与抽帧。

## 3. 实现方案细化 (Implementation Roadmap)

### 第一阶段：数据协议定义 (W1)
开发组首先要定义 **Canvas JSON Schema**。一个典型的节点应包含：
```json
{
  "id": "node_001",
  "type": "T2I_Node",
  "data": { "prompt": "...", "seed": -1, "model": "SDXL" },
  "inputs": { "ref_image": "node_000_output" },
  "status": "processing"
}
```

### 第二阶段：ComfyUI 封装与解耦 (W2-W3)
不要直接在前端写 ComfyUI 的逻辑，而是后端做一个 **Workflow Adapter**：
1.  预设几套标准的 ComfyUI 工作流（Json 模板）。
2.  后端根据前端画布的连线，动态填入参数，生成最终的 `prompt_api.json` 发送给 ComfyUI。
3.  *关键技术：* **IP-Adapter-FaceID** 节点封装，确保角色一致性。

### 第三阶段：画布与状态机联调 (W4-W5)
1.  实现“拓扑排序”算法：点击最后的“视频导出”时，自动计算整个图的执行顺序。
2.  **缓存机制：** 如果上游节点参数没变，再次执行时直接复用缓存结果，节省 GPU。

## 4. 关键挑战与解决方案 (Pitfalls & Solutions)

| 挑战 | 解决方案 |
| :--- | :--- |
| **显存溢出 (OOM)** | 实现后端 **GPU 资源池管理**。限制每个用户同时运行的节点数，超出则进入 Redis 队列排队。 |
| **角色长相走样** | 引入 **ReActor (换脸)** 节点或 **InstantID**。在视频生成的最后一步加一层面部修复。 |
| **画布性能卡顿** | 采用 **Canvas Layer 虚拟化**。在大规模故事板（100+ 节点）时，非视口区域不进行 React 组件更新。 |
| **多用户并发** | 配合你的 **AI Private Hub**，支持“本地执行”和“云端执行”切换，减轻服务器压力。 |

## 5. 验收清单 (The "Done" List)

1.  **[ ] 基础流：** 能否从一张参考图出发，经过 3 个不同 Prompt 的节点，生成风格统一的 3 张分镜？
2.  **[ ] 视频流：** 节点能否正确接收图片并调用 SVD 或 Luma 模型转为视频？
3.  **[ ] 稳定性：** 连续生成 20 组分镜，后端是否会出现任务堆积或内存泄漏？
4.  **[ ] 易用性：** 非技术用户是否能在不了解 ComfyUI 连线逻辑的情况下，通过你的 UI 完成短剧编排？
