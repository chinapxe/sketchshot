# SketchShot 正式验收清单

> 更新时间：2026-04-04 09:16:21 +08:00
> 文档定位：M5 收尾阶段正式验收与交付验收执行清单

---

## 1. 验收目标

本清单用于确认 SketchShot 已达到以下状态：

- 主链路可运行
- 文档与部署方式一致
- 在线 Docker 运行可复现
- 离线打包与目标机部署可复现
- 关键业务场景可演示、可交付、可回归

---

## 2. 验收范围

本次正式验收建议至少覆盖以下四类内容：

1. 代码级基础验证
2. 源机器在线 Docker 验证
3. 离线打包与目标机部署验证
4. 关键业务链路人工验收

---

## 3. 验收前准备

执行前先确认：

- 当前代码目录完整，建议工作区干净
- 源机器已安装并启动 Docker Desktop
- 目标机器已安装并启动 Docker Desktop 或 Docker Engine
- 如需真实火山引擎，源机器与目标机器都可访问 `ARK_BASE_URL`
- 如需真实火山引擎，源机器 `.env.docker` 中已配置 `WXHB_VOLCENGINE_ENABLED=True` 和 `WXHB_ARK_API_KEY`
- 如需真实火山引擎，目标机器离线包 `.env` 中已配置 `VOLCENGINE_ENABLED=True` 和 `ARK_API_KEY`

---

## 4. 代码级基础验证

### 4.1 后端验证

执行命令：

```powershell
python -m unittest discover -s backend/tests -p "test_*.py"
```

通过标准：

- 所有测试通过
- 无导入错误
- 无明显环境变量解析错误

### 4.2 前端验证

执行命令：

```powershell
Set-Location .\frontend
npm test
npm run build
```

通过标准：

- 单元测试全部通过
- 生产构建成功
- 如存在 chunk warning，可记录为后续优化项，但不应阻断本次验收

---

## 5. 源机器在线 Docker 验证

### 5.1 启动服务

执行命令：

```powershell
.\scripts\docker-build.ps1 -Action up
```

通过标准：

- backend 容器启动成功
- frontend 容器启动成功
- `docker compose ps` 中 backend 为 `healthy`

### 5.2 健康检查

检查地址：

- 前端：`http://localhost:8080/`
- 后端健康检查：`http://localhost:8000/api/health`

通过标准：

- 前端页面可以正常打开
- 后端健康检查返回 `status=ok`
- 如计划走火山引擎主链路，健康检查结果应体现 `default_adapter=volcengine`

### 5.3 在线基础操作

人工检查：

- 可以打开画布
- 可以新增节点
- 可以拖拽和连线
- 推荐模板可以正常套用

通过标准：

- 无首屏报错
- 无明显中英文混杂的核心操作入口
- 无无法继续操作的阻断性 UI 异常

---

## 6. 离线打包验证

### 6.1 生成离线包

执行命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1
```

通过标准：

- 成功生成 `offline-bundle\时间戳\`
- 成功生成 `offline-bundle\时间戳.zip`
- 生成 `wxhb-images-offline.tar`
- 生成 `wxhb-images-offline.sha256.txt`

### 6.2 离线包文件完整性

检查以下关键文件是否存在：

- `.env`
- `.env.example`
- `docker-compose.offline.yml`
- `Deploy-OfflineBundle.ps1`
- `Verify-OfflineBundle.ps1`
- `Cleanup-OfflineBundle.ps1`
- `Start-SketchShot.cmd`
- `Verify-SketchShot.cmd`
- `Cleanup-SketchShot.cmd`
- `README_OFFLINE.txt`
- `data\`

通过标准：

- 关键文件齐全
- `.env` 中端口、适配器和 API Key 配置符合预期

---

## 7. 目标机器部署验证

### 7.1 目标机器部署

执行方式：

- 双击 `Start-SketchShot.cmd`

或执行命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\Deploy-OfflineBundle.ps1
```

通过标准：

- 离线镜像导入成功
- backend、frontend 容器启动成功
- 脚本结束时输出访问地址

### 7.2 目标机器自动校验

执行方式：

- 双击 `Verify-SketchShot.cmd`

或执行命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\Verify-OfflineBundle.ps1
```

通过标准：

- backend 容器为 `running` 且 `healthy`
- frontend 容器为 `running` 且 `healthy`
- 后端健康检查通过
- 前端页面访问通过

### 7.3 目标机器人工复查

检查地址：

- 前端：`http://localhost:8080/`
- 后端健康检查：`http://localhost:8000/api/health`

通过标准：

- 前端页面正常加载
- 能进入主界面
- 无资源缺失或空白页

---

## 8. 关键业务链路验收

### 8.1 故事板模板链路

建议使用推荐模板之一进行验收：

- `推荐 · 一场戏三镜头`
- `推荐 · 角色设定到镜头`
- `推荐 · 连续动作视频镜头`

通过标准：

- 模板可以正常套用到画布
- 节点关系与模板说明一致
- 用户可以理解“场次 / 角色 / 风格 / 镜头”的主链路

### 8.2 本地草稿恢复

操作步骤：

1. 在画布中创建或修改一个项目
2. 刷新浏览器页面

通过标准：

- 当前画布内容可恢复
- 节点和连线不丢失
- 关联的本地资源不会整体失效

### 8.3 导入导出

操作步骤：

1. 导出当前项目
2. 在空白画布重新导入

通过标准：

- 可导出为本地文件包
- 可重新导入
- 节点、连线、资源引用基本恢复正确
- 历史 `wxhb` / 新 `sketchshot` 格式兼容不报错

### 8.4 图像生成链路

操作步骤：

1. 选择一个图像镜头或图片生成节点
2. 提交图像生成

通过标准：

- 能正确进入排队 / 处理中状态
- WebSocket 进度可以更新
- 成功后图像结果落到画布和输出节点

### 8.5 视频生成链路

操作步骤：

1. 选择一个视频镜头或视频生成节点
2. 提交视频生成

通过标准：

- 能正确进入排队 / 处理中状态
- 超时策略可覆盖较长视频生成
- 成功后视频结果落到画布和输出节点

### 8.6 提示词优化链路

操作步骤：

1. 在图像或视频节点中点击 `AI Refine`

通过标准：

- 能正常调用后端提示词生成接口
- 返回内容能回填到当前节点

---

## 9. 异常场景验收

建议至少检查以下异常情况：

- 未配置 API Key 时，系统提示是否明确
- 后端未启动时，前端提示是否明确
- 导入损坏项目包时，错误提示是否明确
- 端口占用时，部署脚本或 Docker 输出是否可定位

通过标准：

- 报错信息能指出问题方向
- 不出现明显无提示失败
- 不出现页面完全卡死且无法恢复的情况

## 11. 当前建议结论

在当前阶段，最关键的一次正式验收不是“代码能不能跑”，而是：

- 交付链路是否一致
- 目标机器是否能复现部署
- 故事板主链路是否能被稳定演示

如果以上三项都通过，就说明 M5 收尾已经进入可交付状态。
