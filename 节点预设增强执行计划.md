## 1. 目标与结论

本次增强的目标，不是把 SketchShot 做成复杂的提示词工程面板，而是：

- 把高价值关键词沉淀为可复用的节点预设
- 让用户在故事板节点里更快搭出“镜头语言骨架”
- 保持画布节点紧凑，不再继续纵向拉长
- 让结构化预设能够真正参与主链路，而不只是展示用标签

一句话原则：

**预设负责快速搭骨架，自由文本负责补细节；结构化增强要服务故事板表达，而不是增加参数负担。**

---

## 2. 本次已确认的执行方向

后续实现按以下方向执行，不再重复讨论大方向：

1. 优先增强 `ShotNode` 与 `StyleNode`
2. `ImageGenNode` 与 `VideoGenNode` 只做轻量快捷模板，不做完整词库面板
3. `CharacterNode` 暂不做大词库，只保留极轻量增强空间
4. UI 采用“节点内轻面板 + 横向分类切换 + 已选摘要”的方式
5. 不使用超长下拉框，不把六大类关键词全部平铺到节点正文
6. 结构化预设必须参与 prompt 拼装、节点摘要显示、模板保存与模板回填

---

## 3. 产品边界

### 3.1 要坚持的边界

- SketchShot 是“无限画布故事板工具”，不是“复杂 AI 工作流平台”
- 节点首先表达创作语义，不是技术参数面板
- 预设要帮助用户快速组织镜头，不是让用户学习提示词工程
- 结构化字段数量要克制，优先高频、高价值、直观可理解的维度

### 3.2 明确不做的方向

- 不做超长多级表单
- 不做完整“提示词词典编辑器”
- 不做把所有关键词都塞进 `ImageGenNode` / `VideoGenNode`
- 不让单个节点高度继续明显增长
- 不在一期做复杂联动推荐、自动补全、权重编辑

## 6. ShotNode 详细改良计划

`ShotNode` 是本次预设增强的核心承载节点。

### 6.1 保留的现有字段

- 镜头标题
- 镜头描述
- 景别
- 机位
- 动作
- 情绪
- 输出类型
- 画面比例
- 分辨率
- 时长
- 运动强度
- 首帧 / 尾帧约束
- 九宫格连续动作
- 补充提示

### 6.2 新增的结构化预设维度

- 运镜语言
- 构图方式
- 光影风格
- 氛围标签
- 质感标签

### 6.3 字段形式建议

- 景别：单选
- 机位：单选
- 运镜：单选，必要时二期放宽到最多双选
- 构图：单选
- 光影：单选
- 氛围：多选，上限 2 到 3 个
- 质感：多选，上限 2 到 3 个

### 6.4 UI 方案

- 不把所有预设直接摊在节点正文
- 在 `ShotNode` 中新增“镜头预设”入口
- 点开后显示一个轻量面板
- 面板顶部采用横向分类切换：
  - 景别
  - 运镜
  - 构图
  - 光影
  - 氛围
  - 质感
- 每个分类内部使用 chips 选择，不用超长下拉
- 节点正文只显示“已选摘要”

### 6.5 折叠态展示要求

折叠状态至少显示：

- 景别
- 运镜
- 构图
- 1 到 2 个氛围标签
- 是否有首尾帧
- 是否已有结果

### 6.6 Prompt 拼装要求

`ShotNode` 结构化预设进入 prompt 时，顺序建议固定为：

1. 场次上下文
2. 角色设定
3. 风格设定
4. 镜头结构化预设
5. 用户自由补充

### 6.7 一期验收口径

- 用户不写太多补充文本，也能靠预设快速搭出一个“像镜头”的镜头节点
- 折叠后依然能快速看懂镜头差异
- 节点高度增加受控，不影响画布阅读

## 7. StyleNode 详细改良计划

`StyleNode` 负责承载全局视觉气质，是本次第二优先级主节点。

### 7.1 保留的现有字段

- 风格名称
- 关键词
- 色彩氛围
- 光线方向
- 镜头语言
- 备注

### 7.2 增强方式

不优先继续新增字段，而是把现有字段改造成：

- 预设 chips 快速填充
- 用户仍可手动补充和编辑文本

### 7.3 UI 方案

- 每个字段下方提供常用 chips
- 点选 chips 后自动写入对应字段
- 已写文本仍允许继续修改
- 预设 chips 不要求单独保存，点击即生效

### 7.4 各字段对应预设来源

- `关键词`
  - 电影级光影
  - 叙事感强
  - 自然真实
  - 高质感
  - 影院级色彩
- `色彩氛围`
  - 冷青灰
  - 暖金棕
  - 霓虹紫蓝
  - 低饱和
- `光线方向`
  - 软光
  - 硬光
  - 侧光
  - 逆光
  - 顶光
  - 低调光影
- `镜头语言`
  - 三分构图
  - 对称构图
  - 留白构图
  - 框架构图
  - 引导线构图
  - 压迫式留白

### 7.5 折叠态展示要求

- 风格名称
- 关键词摘要
- 色彩标签
- 光线标签
- 是否已补镜头语言

### 7.6 一期验收口径

- 用户能在 10 秒内快速搭出一套风格约束
- 风格节点内容能明显区分不同模板和不同镜头分支

## 8. ImageGenNode 详细改良计划

`ImageGenNode` 不作为完整词库承载节点，只做轻量快捷增强。

### 8.1 增强目标

- 降低新手第一次出图的思考成本
- 提供少量“直接可用”的画面模板
- 不把节点重新做成复杂参数面板

### 8.2 建议形态

- 顶部或 prompt 区右上角提供“快捷模板”入口
- 一期内置 3 个模板：
  - 情绪特写
  - 叙事中景
  - 史诗大场景

### 8.3 模板作用方式

- 点击模板后，将预设短语补入 prompt
- 不新增长表单字段
- 用户可继续修改 prompt

### 8.4 可选的轻量补充

二期可再考虑加入一行轻量 chips：

- 中心构图
- 三分构图
- 留白构图
- 柔和光线
- 电影颗粒

### 8.5 一期验收口径

- 用户第一次进入图片节点时，不需要空想 prompt
- 一键套用模板即可得到有明显风格倾向的基础结果

---

## 9. VideoGenNode 详细改良计划

`VideoGenNode` 重点不在更多技术参数，而在“运镜语言表达”。

### 9.1 增强目标

- 帮助用户更容易写出可执行的视频运动描述
- 用少量快捷模板解决“不会描述镜头怎么动”的问题

### 9.2 建议形态

- 增加 3 个视频快捷模板：
  - 稳定镜头
  - 电影推镜
  - 紧张手持

### 9.3 可加的轻量 chips

- 固定镜头
- 推镜
- 拉镜
- 跟镜
- 环绕
- 手持
- 轻微抖动

### 9.4 约束

- 不把构图、光影、氛围整套词库搬到视频节点
- 视频节点仍以“运动描述”文本为主

### 9.5 一期验收口径

- 用户可以更快组织出像样的视频描述
- 不需要理解专业术语也能从模板开始

---

## 10. CharacterNode 详细改良计划

`CharacterNode` 暂不做大词库节点。

### 10.1 当前定位

- 角色设定主节点
- 稳定外观与参考图
- 为镜头与出图节点提供参考身份

### 10.2 可保留的轻量增强空间

二期如需增强，只考虑少量标签：

- 气质：冷静、锋利、温柔、脆弱、疲惫、压抑
- 状态：湿发、奔跑后、狼狈、整洁、盛装、受伤

### 10.3 当前明确不做

- 不做大规模人物词库
- 不做人设百科表单
- 不让角色节点变成超长人物资料卡

---

## 11. 共享 UI 规则

### 11.1 总体规则

- 默认收起详细预设
- 已选项以摘要形式显示
- 详细分类通过横向切换查看
- 优先使用 chips，不优先使用下拉
- 节点宽度变大时自动允许 chips 换行

### 11.2 交互规则

- 点选即生效，不增加二次保存步骤
- 同类互斥项使用单选
- 可叠加项使用多选
- 多选项设置上限，避免 prompt 过长
- 必须支持一键清空当前分类

### 11.3 画布可读性要求

- 节点在默认宽度下不应显著变高
- 折叠态摘要必须足够表达差异
- 节点宽度被手动拉宽后，摘要布局应更自然

## 12. 模板体系联动计划

`workflowTemplates.ts` 后续应支持保存结构化预设，而不只保存纯文本 prompt。

### 12.1 模板侧要实现的能力

- 模板保存 `ShotNode` 预设字段
- 模板保存 `StyleNode` 预设组合
- 模板卡片展示“镜头类型 / 风格摘要 / 适用方向”

### 12.2 模板展示建议

模板卡片中建议增加：

- 适合类型：情绪特写 / 叙事中景 / 史诗大场景 / 视频运镜
- 预设标签摘要
- 学习点

### 12.3 模板应用后的要求

- 节点里能直接看见预设摘要
- 用户能在模板基础上继续点选或清空预设

---

## 13. Prompt 拼装与数据策略

### 13.1 Prompt 拼装要求

- 结构化预设必须参与实际 prompt
- 结构化内容要做去重与限量
- 用户自由输入优先级仍然最高
- 上游 `StyleNode` 负责全局气质
- 当前 `ShotNode` 负责局部镜头语言

### 13.2 数据结构要求

后续实现时，新增字段建议采用“明确枚举 + 可空数组”模式，不直接塞到长文本里。

例如：

- `cameraMovement`
- `composition`
- `lightingStyle`
- `moodTags`
- `qualityTags`

### 13.3 缓存与签名要求

- 新增预设字段必须进入生成签名
- 结构化预设变化后，应正确触发“需更新”

## 14. 分期实施顺序

### 第一阶段：词库与字段设计

- 确定一期词库范围
- 确定每个节点承接哪些预设维度
- 确定结构化字段命名

### 第二阶段：`ShotNode` 与 `StyleNode`

- 先做 UI 结构
- 再接 prompt 拼装
- 再接折叠态摘要

### 第三阶段：模板联动

- 模板保存与回填预设
- 模板卡片显示摘要

### 第四阶段：`ImageGenNode` / `VideoGenNode`

- 加快捷模板
- 加轻量 chips

### 第五阶段：复盘与扩展

- 评估是否补二期长尾词
- 评估是否需要轻量增强 `CharacterNode`

---

## 15. 一期验收标准

一期完成后，应至少满足以下标准：

1. 新用户可以在 `ShotNode` 中通过预设快速搭出镜头骨架
2. `StyleNode` 可以通过预设快速建立统一风格
3. 节点默认高度可控，不造成新的滚动负担
4. 折叠态摘要能帮助用户在画布中快速识别不同节点
5. 模板应用后，预设字段能够正确显示并参与生成
6. prompt 拼装逻辑不会因结构化预设导致明显冗余或冲突
