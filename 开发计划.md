## 一、项目定位

以 **无限画布** 为核心的 AI 创作工具。用户在画布上通过拖拽节点、连线编排，完成图片/视频的 AI 生成工作流。后端模型层为可插拔适配架构，不绑定特定 AI 引擎。

---

## 二、技术选型

| 层级 | 技术栈 | 说明 |
|------|--------|------|
| 构建工具 | Vite 6 | 极速 HMR，开发体验好 |
| 前端框架 | React 18 | 生态成熟，组件化 |
| 画布引擎 | React Flow | 开源无限画布，节点/连线系统完善 |
| 状态管理 | Zustand | 轻量高效，比 XState 更适合画布场景 |
| UI 组件库 | Ant Design 5 | 节点内表单控件丰富 |
| 样式方案 | TailwindCSS 4 | 原子化 CSS，快速布局 |
| 实时通信 | Socket.io-client | 接收后端生成进度推送 |
| 后端框架 | FastAPI | 异步高性能 Python 框架 |
| 任务队列 | Redis Streams | 轻量级任务分发 |
| 数据库 | PostgreSQL | 工作流持久化存储 |
| 文件存储 | MinIO / 本地 | 图片视频资源存储 |

## 三、开发阶段划分

### 第一阶段：项目骨架与画布基础（Week 1）

| 序号 | 任务 | 交付物 |
|------|------|--------|
| 1.1 | 初始化前端项目（Vite + React + TypeScript） | 可运行的空项目 |
| 1.2 | 集成 React Flow，实现无限画布基础 | 画布支持缩放、平移、网格背景 |
| 1.3 | 设计并实现节点类型系统（TypeScript 类型定义） | 节点/连线的数据结构定义 |
| 1.4 | 实现「图片上传节点」 | 可拖入画布，支持图片上传和预览 |
| 1.5 | 实现「图片生成节点」 | 包含提示词、比例、分辨率、参考图、生成按钮 |
| 1.6 | 实现「图片展示节点」（输出节点） | 展示生成结果，支持点击放大 |
| 1.7 | 实现节点间智能连线 | 类型检查，Image→Image 合法，不合法连线阻止 |

### 第二阶段：画布交互增强（Week 2）

| 序号 | 任务 | 交付物 |
|------|------|--------|
| 2.1 | 连线数据传递 — 参考图沿连线自动填充到下游节点 | 上游图片自动出现在下游「参考图」区域 |
| 2.2 | 节点克隆、删除、禁用/启用 | 右键菜单或工具栏操作 |
| 2.3 | 节点状态系统（空闲/排队/生成中/成功/失败） | 节点边框颜色/图标随状态变化 |
| 2.4 | 参数热更新提示 — 上游参数变更时下游显示"待更新" | 视觉提示 + 重新生成按钮 |
| 2.5 | 画布工具栏（撤销/重做/全选/自动布局/缩放控件） | 顶部或侧边工具栏 |
| 2.6 | 节点侧边栏 — 可拖拽的节点面板 | 左侧节点类型列表，拖入画布添加 |
| 2.7 | 小地图（MiniMap）与画布导航 | 右下角缩略图导航 |

### 第三阶段：后端基础与前后端联调（Week 3）

| 序号 | 任务 | 交付物 |
|------|------|--------|
| 3.1 | 初始化后端项目（FastAPI + 项目结构） | 可运行的后端服务 |
| 3.2 | 设计 RESTful API 协议（工作流 CRUD、生成任务） | API 文档（OpenAPI） |
| 3.3 | 实现工作流保存/加载 API | 画布 JSON 持久化到数据库 |
| 3.4 | 实现模型适配器接口（Adapter 抽象层） | 统一的模型调用接口定义 |
| 3.5 | 实现 Mock 适配器（返回模拟图片） | 前端可跑通完整流程 |
| 3.6 | WebSocket 进度推送 | 生成进度实时推送到前端节点 |
| 3.7 | 前后端联调 — 画布生成按钮触发后端任务 | 点击生成 → 后端处理 → 结果回显到画布 |

### 第四阶段：高级功能与体验优化（Week 4-5）

| 序号 | 任务 | 交付物 |
|------|------|--------|
| 4.1 | 拓扑排序 — 一键执行整个工作流 | 自动按依赖顺序逐节点执行 |
| 4.2 | 缓存机制 — 参数未变时复用上次结果 | 节省重复计算 |
| 4.3 | 积分系统 UI（💎消耗提示） | 生成按钮显示消耗积分 |
| 4.4 | 画布性能优化 — 虚拟化渲染（100+ 节点不卡） | 视口外节点不渲染 |
| 4.5 | 图片/视频预览弹窗（高清查看） | 点击节点内图片弹出大图 |
| 4.6 | 工作流模板系统 — 预设常用工作流 | 用户可一键加载模板 |
| 4.7 | 导出功能 — 批量导出生成结果 | 打包下载所有输出图片/视频 |

### 第五阶段：真实模型接入与上线准备（Week 6+）

| 序号 | 任务 | 交付物 |
|------|------|--------|
| 5.1 | 接入第一个真实 AI 模型（如 ComfyUI / SD API） | 真实图片生成 |
| 5.2 | 角色一致性方案集成（IP-Adapter / InstantID） | 参考图人物特征保持 |
| 5.3 | 视频生成节点实现 | 图片转视频能力 |
| 5.4 | 多用户与权限管理 | 登录、项目隔离 |
| 5.5 | 部署方案（前端 Nginx + 后端 Docker） | 生产环境部署脚本 |
| 5.6 | 压力测试与稳定性验证 | 连续生成 20 组不崩溃 |

## 五、关键设计原则

1. **API 地址不硬编码** — 前端通过 `.env` 配置文件管理 API 请求地址，开发/生产环境统一切换
2. **模型可插拔** — 后端通过 Adapter 模式对接不同 AI 模型，新增模型只需实现适配器接口
3. **画布优先** — 先用 Mock 数据打通画布全流程，再逐步接入真实模型
4. **渐进增强** — 每个阶段交付可运行的版本，持续迭代
