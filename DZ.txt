=========================================================================

我参考了截至 2026-04-01 可公开查到的同类产品官方信息，结合你们当前“主链路已打通”的状态，判断下一步最值得做的不是再扩底层接入，而是把产品从“能跑”提升到“更好用、更可复用、更像正式工具”。

**建议增强方向**
- `资产中心`：做项目级素材库、输出库、文件夹、标签、收藏、版本对比、分享链接。这个方向很像 Runway 的资产库和 Krea 的资产管理能力。
- `工作流模板库`：把常用流程沉淀成模板，并支持复制、分享、导入导出。ComfyUI 和 Runway 都把模板/工作流库当成核心效率入口。
- `节点文档与引导`：给每个节点增加说明、参数解释、示例输入输出、常见错误。ComfyUI 现在已经把 node docs 做成内置能力，这对接手和上手很关键。
- `视频提示词助手 2.0`：不要只做“润色 prompt”，而是拆成“主体动作、环境运动、镜头运动、节奏/时序、方向/速度”几个结构化输入。Runway 的图生视频提示词指南就是这么组织的。
- `镜头级视频节点`：加入“首帧/尾帧”“连续镜头”“循环镜头”“镜头拼接”“从上一段末帧续生成”。这类能力对视频工作流非常重要，Runway 和 Pika 都在强化这块。
- `执行中心`：增加 Active Runs/任务队列/取消重试/失败原因/历史记录/耗时统计。Runway 和 fal.ai 都很强调多步骤运行中的可观测性。
- `批量编辑与冻结输出`：支持多选节点统一改参数，支持锁定节点输出避免重复生成。Runway 的 batch edit 和 lock node 都很实用。
- `角色圣经 / 风格包`：把角色参考图、风格参考、默认提示词规范、画幅、输出规格沉淀成项目级配置，类似 Canva 的 Brand Kit 思路，但更偏“角色一致性”和“视觉风格一致性”。
- `输出交付层`：除了预览，还要有“导出成项目包”“一键打包全部图片/视频”“按镜头顺序导出”“附带生成参数”。这会直接提升项目可交付性。
- `协作层`：后面可以做工作区、成员、评论、审批状态、共享工作流，但这建议放在第二梯队，不要抢当前主线。

**我建议的下一步计划**
- `第一阶段（最推荐，先做产品闭环）`：资产中心、执行中心、输出版本对比、视频提示词助手 2.0。
- `第二阶段（把“视频工具感”做出来）`：首尾帧节点、连续镜头续生成、循环镜头、镜头拼接、更多视频专用模板。
- `第三阶段（把“团队可复用”做出来）`：模板库、节点文档、角色圣经/风格包、导入导出、分享。
- `第四阶段（再考虑协作）`：工作区、成员权限、评论审核、团队资产共享。

**如果只选 3 个最该先做**
- `资产中心`
- `视频提示词助手 2.0`
- `执行中心`

这三个做完，产品体验会立刻从“技术演示版”变成“可持续创作工具”。

**暂时不建议优先做**
- 深度 ComfyUI 对接
- 多租户/计费系统
- 复杂企业权限
- 过早做太多模型接入

这些都不是当前最短路径。
