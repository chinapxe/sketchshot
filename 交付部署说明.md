# SketchShot 交付部署说明

> 更新时间：2026-04-02 22:55:49 +08:00
> 文档定位：项目交付、Docker 运行、离线打包、目标机部署操作说明

---

## 1. 文档用途

本文档用于说明：

- 源机器如何构建并打包 SketchShot
- 如何生成可分发的离线交付包
- 目标机器如何一键部署运行
- `scripts` 目录下各脚本的作用和适用场景

当前项目已经支持两种使用方式：

- 本机 Docker 运行
- 生成离线交付包后，在目标机器上离线部署运行

---

## 2. 相关文件说明

项目根目录中与交付部署直接相关的文件如下：

- `docker-compose.yml`
  本机 Docker 构建与运行编排文件

- `docker-compose.offline.yml`
  离线交付包内使用的部署编排文件

- `.env.docker`
  本机 Docker 构建与运行时使用的环境配置

- `.env.docker.example`
  本机 Docker 环境变量模板

- `.env.offline.example`
  离线包环境变量模板

说明：

- 离线打包时，会尽量读取当前 `.env.docker` 中已有的有效配置，并生成离线包里的 `.env`
- 如果 `.env.docker` 中没有填写某些值，离线包会保留 `.env.offline.example` 的默认值
- 当前产品正式名已切换为 `SketchShot`，但环境变量前缀如 `WXHB_ARK_API_KEY` 仍暂时保留用于兼容

## 3. 前置条件

源机器打包前需要满足：

- 已安装并启动 Docker Desktop
- 当前项目代码完整
- 如果需要真实调用火山引擎，`.env.docker` 中应配置好 `WXHB_ARK_API_KEY`
- 若构建依赖下载较慢，可在 `.env.docker` 中配置国内 PyPI 镜像

目标机器部署前需要满足：

- 已安装并启动 Docker Desktop 或 Docker Engine
- 能正常执行 `docker` 和 `docker compose`
- 已拿到打包生成的离线交付目录或压缩包

---

## 4. 推荐交付流程

标准交付流程分为两段：

1. 源机器打包
2. 目标机器部署

### 4.1 源机器打包

在项目根目录 `C:\2026\WXHB` 执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1
```

或直接双击：

```text
scripts\Build-OfflineBundle.cmd
```

执行后会自动完成：

- 构建前后端 Docker 镜像
- 将镜像保存为离线 tar 包
- 复制离线部署所需的 compose、env、部署脚本、校验脚本、清理脚本
- 生成目录 `offline-bundle\时间戳\`
- 同时生成一个可直接分发的压缩包 `offline-bundle\时间戳.zip`

如果镜像已经构建好，只想重新生成离线包，可以执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1 -SkipBuild
```

如果希望强制重新构建镜像，可执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1 -NoCache
```

### 4.2 交付给目标机器

将以下任意一种交付给目标机器：

- `offline-bundle\时间戳\` 整个目录
- `offline-bundle\时间戳.zip` 压缩包

推荐直接交付 zip，由目标机器解压后部署。

### 4.3 目标机器部署

Windows 目标机器推荐直接双击：

```text
Start-SketchShot.cmd
```

等价命令为：

```powershell
powershell -ExecutionPolicy Bypass -File .\Deploy-OfflineBundle.ps1
```

执行后会自动完成：

- 导入离线镜像 `wxhb-images-offline.tar`
- 使用 `docker-compose.offline.yml` 启动前后端容器
- 自动执行健康检查
- 输出访问地址

默认访问地址通常为：

- 前端：`http://localhost:8080/`
- 后端健康检查：`http://localhost:8000/api/health`

如果需要调整端口或火山引擎配置，可在部署前先修改离线包里的 `.env`。

### 4.4 目标机器部署后校验

Windows 目标机器可双击：

```text
Verify-SketchShot.cmd
```

等价命令为：

```powershell
powershell -ExecutionPolicy Bypass -File .\Verify-OfflineBundle.ps1
```

该脚本会校验：

- backend 容器是否 `running` 且 `healthy`
- frontend 容器是否 `running` 且 `healthy`
- 后端健康接口是否返回正常
- 前端页面是否可以正常访问

### 4.5 目标机器清理

Windows 目标机器可双击：

```text
Cleanup-SketchShot.cmd
```

等价命令为：

```powershell
powershell -ExecutionPolicy Bypass -File .\Cleanup-OfflineBundle.ps1
```

该操作会：

- 停止并删除离线部署容器
- 删除离线镜像标签
- 默认保留 `data` 目录中的运行数据

如果需要彻底清理运行数据，可执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\Cleanup-OfflineBundle.ps1 -PurgeData
```

## 5. scripts 目录文件作用说明

### 5.1 Windows 打包与本机运行脚本

- `scripts\docker-build.ps1`
  本机 Docker 运行管理脚本。用于 `build`、`up`、`down`、`restart`、`logs`、`ps`、`config` 等操作，主要服务于开发机或演示机本地运行，不是离线交付包内的主入口。

- `scripts\Package-OfflineBundle.ps1`
  离线交付包核心打包脚本。负责构建镜像、离线镜像打包、生成离线目录、复制部署脚本、生成 `.env`、生成 zip 压缩包。

- `scripts\Build-OfflineBundle.cmd`
  Windows 双击版离线打包入口。内部实际调用 `Package-OfflineBundle.ps1`，适合不想手敲命令时使用。

### 5.2 Windows 目标机部署脚本

- `scripts\Deploy-OfflineBundle.ps1`
  离线包部署主脚本。负责加载镜像、启动容器、调用校验脚本、输出访问地址。

- `scripts\Verify-OfflineBundle.ps1`
  离线部署校验脚本。负责检查容器状态、健康检查结果、前端访问结果。

- `scripts\Cleanup-OfflineBundle.ps1`
  离线部署清理脚本。负责停止容器、删除离线镜像标签，并可选清理数据目录。

- `scripts\Start-SketchShot.cmd`
  Windows 双击版部署入口。内部调用 `Deploy-OfflineBundle.ps1`。

- `scripts\Verify-SketchShot.cmd`
  Windows 双击版校验入口。内部调用 `Verify-OfflineBundle.ps1`。

- `scripts\Cleanup-SketchShot.cmd`
  Windows 双击版清理入口。内部调用 `Cleanup-OfflineBundle.ps1`。

### 5.3 Linux 目标机部署脚本

- `scripts\Deploy-OfflineBundle.sh`
  Linux 目标机器部署脚本，对应 Windows 版 `Deploy-OfflineBundle.ps1`。

- `scripts\Verify-OfflineBundle.sh`
  Linux 目标机器校验脚本，对应 Windows 版 `Verify-OfflineBundle.ps1`。

- `scripts\Cleanup-OfflineBundle.sh`
  Linux 目标机器清理脚本，对应 Windows 版 `Cleanup-OfflineBundle.ps1`。

## 6. 常用命令速查

### 6.1 本机 Docker 运行

构建镜像：

```powershell
.\scripts\docker-build.ps1 -Action build
```

启动服务：

```powershell
.\scripts\docker-build.ps1 -Action up
```

查看状态：

```powershell
.\scripts\docker-build.ps1 -Action ps
```

查看日志：

```powershell
.\scripts\docker-build.ps1 -Action logs
```

停止服务：

```powershell
.\scripts\docker-build.ps1 -Action down
```

### 6.2 离线打包

标准打包：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1
```

跳过构建直接打包：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1 -SkipBuild
```

无缓存重建后打包：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1 -NoCache
```

### 6.3 Windows 目标机部署

部署：

```powershell
powershell -ExecutionPolicy Bypass -File .\Deploy-OfflineBundle.ps1
```

校验：

```powershell
powershell -ExecutionPolicy Bypass -File .\Verify-OfflineBundle.ps1
```

清理：

```powershell
powershell -ExecutionPolicy Bypass -File .\Cleanup-OfflineBundle.ps1
```

清理并删除运行数据：

```powershell
powershell -ExecutionPolicy Bypass -File .\Cleanup-OfflineBundle.ps1 -PurgeData
```

## 7. 离线包目录结构说明

离线打包完成后，生成目录大致如下：

```text
offline-bundle\20260402_xxxxxx\
  .env
  .env.example
  docker-compose.offline.yml
  wxhb-images-offline.tar
  wxhb-images-offline.sha256.txt
  Deploy-OfflineBundle.ps1
  Verify-OfflineBundle.ps1
  Cleanup-OfflineBundle.ps1
  Start-SketchShot.cmd
  Verify-SketchShot.cmd
  Cleanup-SketchShot.cmd
  Start-WXHB.cmd
  Verify-WXHB.cmd
  Cleanup-WXHB.cmd
  Deploy-OfflineBundle.sh
  Verify-OfflineBundle.sh
  Cleanup-OfflineBundle.sh
  README_OFFLINE.txt
  data\
```

说明：

- `.env` 是可直接运行的目标机配置
- `.env.example` 是参考模板
- `wxhb-images-offline.tar` 是前后端镜像离线包
- `data\` 是目标机器上的本地持久化目录

## 8. 交付时建议重点确认

正式交付前，建议至少确认以下事项：

- `.env.docker` 中端口配置是否符合预期
- 如果目标机器要使用真实火山引擎，确认 `ARK_API_KEY` 已写入离线包 `.env`
- 源机器已成功生成 zip 包
- 目标机器 Docker 已正常运行
- 目标机器未被其他服务占用 `8000` 或 `8080` 端口

---

## 9. 一句话操作指南

如果只记一套最短流程，可以按下面理解：

1. 源机器执行  
   `powershell -ExecutionPolicy Bypass -File .\scripts\Package-OfflineBundle.ps1`

2. 把生成的 `offline-bundle\时间戳.zip` 发给目标机器

3. 目标机器解压后，双击 `Start-SketchShot.cmd`

4. 如需复查，双击 `Verify-SketchShot.cmd`

5. 如需卸载，双击 `Cleanup-SketchShot.cmd`
